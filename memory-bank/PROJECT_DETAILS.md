# Feedlite - AI 智能驱动个人 RSS 阅读器 (PRD v2.4)

## 一、 项目概述
开发一款部署在公网 VPS 上的轻量级、单用户 RSS 信息流阅读器。系统通过定时抓取主流媒体 RSS，结合 AI 驱动的“多级过滤与推荐打分”机制，精准提取高价值资讯。最后在极简的 Web 前端以无限滚动和支持全文检索的数据流形式展示。项目采用全 Docker 化部署，支持全局代理配置，追求极致的资源利用率与系统鲁棒性。

## 二、 技术栈选型
* **后端框架**：Python 3.x + FastAPI (提供 RESTful API 与自动文档)
* **RSS 解析**：`feedparser` (辅以强健的数据清洗与异常处理逻辑)
* **定时任务**：APScheduler (集成在后端服务中)
* **数据库**：SQLite (启用 FTS5 扩展支持全文检索，**强制开启 WAL 模式**以支持高并发读写)
* **前端展示**：原生 HTML / JavaScript / CSS (单页面应用 SPA，基于 **CSS Variables** 实现无缝主题切换)
* **部署环境**：Docker + Docker Compose

## 三、 核心工作流 (五步流水线设计)
为最大程度降低大模型 API 成本并提升系统性能，核心抓取与处理流程严格遵循以下 5 步：

1. **定时抓取与标准化**：
   * 根据设定时间间隔轮询订阅源。
   * 提取 `<title>`、`<link>`、`<description>` 及 `<content>` (正文截取短文本，为未来扩展备用)。
   * **时间标准化**：将所有解析到的发布时间强制转换为标准 UTC 时间。针对缺失时间或残缺 HTML 标签进行兜底与清洗。
2. **第一道防线 (时间过滤)**：
   * 基于 UTC 时间对比，直接丢弃发布时间早于抓取窗口（如 12 小时前）的老文章。
3. **第二道防线 (毫秒级查重)**：
   * 将经过时间过滤的文章链接 (`link`) 与 SQLite 数据库进行比对。依赖数据库的 `UNIQUE` 唯一索引进行查重，**仅全新未收录的链接才会进入下一步**，避免 AI 重复打分消耗 Token。
4. **AI 过滤与打分 (三级漏斗机制)**：
   * **批量请求**：将全新文章（例如每 50 篇为一组）打包发送给大模型，采用单次请求批量处理。
   * **漏斗 1 (偏好匹配)**：首要根据用户动态添加的 `Tags` (最高权重，如“伊朗战争”) 进行初筛，其次结合 `基础用户画像` 进行匹配。
   * **漏斗 2 (去噪过滤)**：AI 判断并直接过滤掉无价值的垃圾新闻或无效简讯。
   * **漏斗 3 (推荐打分)**：对经过前两轮筛选剩余的“有价值新闻”，评估其推荐强度，并给出 0-100 的最终打分。返回严格映射 ID 的 JSON 结果。
5. **落库与源管理**：
   * 将打分达标的文章正式存入 SQLite。
   * **源熔断与降级**：记录每个 RSS 源的 `error_count`。若连续抓取失败 5 次，自动标记为“休眠”状态，避免阻塞全局任务。

## 四、 画像管理与数据库设计
系统采用单用户极简表结构，支持双模型解耦及动态配置：
* **动态画像与标签 (`users` 表)**：集中管理用户鉴权、`base_prompt` (基础画像) 以及 `active_tags` (动态话题)。用户可在前端自由添加或删除短期关注的 Tag，即时影响 AI 过滤权重。
* **双模型动态配置 (`app_config` 表)**：支持新闻过滤打分和个人画像生成使用不同的 AI 模型及供应商。API Key、模型名称、接口地址直接存入数据库，无需重启即可动态切换。
* **软删除机制**：文章和订阅源的删除操作均为更改 `status` 为 `deleted`，实现前端隐藏，保证数据完整性及搜索索引的稳定。

## 五、 页面与交互设计
* **安全鉴权**：单用户密码/Token 登录校验。
* **极简信息流**：
  * **展示要素**：仅展示标题、简介、分类图标、发布时间 (正文 `content` 仅做后端留存，前端暂不展示)。
  * **交互操作**：点击标题直达原文外链；底部提供 👍 / 👎 按钮。
  * **加载方式**：无限滚动（Infinite Scroll），每次请求 20 条。
* **全局功能**：顶部全局搜索框（基于 SQLite FTS5）；黑白主题切换按钮。
* **订阅源管理 (带预览与分类)**：
  * 支持 5 大精简分类：时政与商业、科技与开发、游戏与文娱、生活与涉猎、综合与其他。
  * 支持添加、状态查看、软删除。**新增预览**：导入新源时，复用后端逻辑抓取前 3 条进行测试展示，确认无误后入库。

## 六、 部署与配置设计
* **静态配置 (`config.yml`)**：集中管理抓取间隔、管理员密码等静态设定。
* **代理配置 (网络兜底)**：针对 VPS IP 可能被 API 供应商（如 Google）针对的问题，支持在环境中配置 HTTP/HTTPS Proxy (如通过环境变量 `HTTP_PROXY` 或在 `config.yml` 中指定代理地址)，确保大模型网络请求的畅通。
* **Docker 封装**：提供 `Dockerfile` 及 `docker-compose.yml`，映射端口并挂载 SQLite 数据库及配置文件的持久化数据卷。

## 七、 项目结构
```
feedlite/
├── app/                        # 后端核心代码 (基于 FastAPI 和异步架构)
│   ├── __init__.py             # Python 包标识
│   ├── main.py                 # FastAPI 应用的主入口：负责启动服务、挂载前端静态页面、注册 API 路由
│   ├── database.py             # 数据库引擎核心：负责配置 SQLAlchemy(Async)、开启 SQLite WAL 模式、执行建库 SQL
│   ├── models.py               # ORM 模型定义：用 Python 类映射数据库表（与 schemas.sql 结构保持一致）
│   ├── routers/                # API 路由层 (Controller)：仅负责接收前端 HTTP 请求和返回数据
│   │   ├── feeds.py            # 信息流 API：获取文章列表、无限滚动分页、FTS5 全文检索、点赞反馈
│   │   └── sources.py          # 订阅源 API：RSS 链接的增删改查、连接测试预览、修改分类
│   └── services/               # 核心业务逻辑层 (Service)：与 HTTP 协议解耦，可独立运行和测试
│       ├── rss_fetcher.py      # 爬虫引擎：负责请求 RSS、清洗 HTML 标签、时间标准化过滤、触发软删除或熔断
│       ├── ai_scorer.py        # AI 打分引擎：负责组装 Prompt、提取动态 Tag、请求大模型 API 并解析 JSON 结果
│       └── scheduler.py        # 定时任务调度器：配置 APScheduler，按周期（如每 12 小时）触发抓取和打分流水线
│
├── static/                     # 前端静态资源 (极简原生 SPA 架构，由 FastAPI 直接代理访问)
│   ├── index.html              # 唯一的 HTML 骨架：包含信息流容器、搜索框、设置面板入口
│   ├── style.css               # 全局样式表：重度依赖 CSS Variables，实现一键黑白主题切换
│   └── app.js                  # 前端交互逻辑：无框架纯原生，处理 API 请求、无限滚动加载、DOM 渲染
│
├── data/                       # 数据库持久化目录 (极其重要，Docker 部署时映射的本地 Volume 目录)
│   ├── feedlite.db             # SQLite 数据库本体 (代码运行时自动生成，包含所有数据和 FTS5 索引)
│   └── schemas.sql             # 咱们刚才写好的 DDL 建库脚本 (由 database.py 在冷启动时读取并执行)
│
├── scripts/                    # 独立测试与运维脚本目录
│   └── test_rss.py             # 之前的抓取测试脚本 (可以在不启动完整 Web 服务的情况下测试某条 RSS 是否正常)
│
├── config.yml                  # 静态配置文件 (代理地址、抓取频率等无需频繁改动的环境变量)
├── requirements.txt            # Python 依赖清单 (fastapi, aiosqlite, sqlalchemy, feedparser, requests 等)
├── Dockerfile                  # 后端运行环境的 Docker 构建说明书
├── docker-compose.yml          # 容器编排文件：定义端口映射、数据卷挂载以及网络代理配置
└── README.md                   # 项目需求说明书与技术架构设计文档
```
